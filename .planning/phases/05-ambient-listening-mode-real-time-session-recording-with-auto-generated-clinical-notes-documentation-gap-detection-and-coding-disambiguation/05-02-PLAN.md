Structure:
1. Standard script header with docstring and usage:
```python
#!/usr/bin/env python3
"""
Pre-compute ambient encounter demo data for the Streamlit UI.

Runs the ambient pipeline on hardcoded transcripts and serializes results
to ui/demo_data/ambient/encounter_NNN.json for instant loading in the
ambient mode page.

Usage:
    python scripts/precompute_ambient.py           # Full pipeline
    python scripts/precompute_ambient.py --quick    # Skip RAG coding (faster)

Note: The handcrafted JSON files in ui/demo_data/ambient/ are the primary
demo artifacts. This script regenerates the pipeline_result portions.
"""
```

2. sys.path setup, logging config (same as precompute_demo.py)

3. Define ENCOUNTERS list with two tuples: `(encounter_id, transcript_text)` where transcript_text is a short representative clinical note (not the full conversation -- just the SOAP note portion, since that is what gets fed to the pipeline). Use the `generated_note` content from the JSON files.

4. For each encounter:
   - Import and call `run_ambient_pipeline` from `cliniq.modules.m6_ambient`

   - Load existing JSON file, update `pipeline_result` and `disambiguation_items` fields with live results
   - Write updated JSON back

5. Support `--quick` flag that passes `skip_coding=True` to run_pipeline_audited (via a parameter plumbed through run_ambient_pipeline -- or just call run_pipeline_audited directly with the SOAP note text).

6. Print summary of processed encounters.

The script should NOT crash if models aren't downloaded -- print a helpful message suggesting demo data files already exist and can be used as-is.

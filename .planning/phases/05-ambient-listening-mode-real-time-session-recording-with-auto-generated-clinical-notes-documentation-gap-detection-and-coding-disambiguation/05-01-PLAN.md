  <action>
Create `cliniq/models/ambient.py` with the following Pydantic v2 models (follow the existing pattern from `cliniq/models/cdi.py` -- BaseModel, Field, computed_field):

1. **EncounterTranscript(BaseModel):**
   - `raw_text: str` -- full conversation transcript
   - `speaker_segments: list[dict] = Field(default_factory=list)` -- optional speaker-labeled segments (for future use)
   - `duration_seconds: float = 0.0` -- audio duration
   - `word_count: int` as a `@computed_field` from `len(raw_text.split())`

2. **StructuredNote(BaseModel):**
   - `chief_complaint: str = ""`
   - `hpi: str = ""` -- History of Present Illness
   - `ros: str = ""` -- Review of Systems
   - `physical_exam: str = ""`
   - `assessment: str = ""`
   - `plan: str = ""`
   - `full_text: str` -- the complete SOAP note as a single string (this is what gets fed to `run_pipeline_audited`)

3. **DisambiguationItem(BaseModel):**
   - `item_id: str` -- unique identifier (uuid4 hex[:8])
   - `category: str` -- one of "gap", "missed_diagnosis", "conflict", "ambiguity"
   - `title: str` -- short display title
   - `description: str` -- detailed explanation
   - `suggested_action: str` -- what the provider should do
   - `source_code: str = ""` -- ICD-10 code involved, if any
   - `confidence: float = Field(default=0.8, ge=0.0, le=1.0)`
   - `status: str = Field(default="pending")` -- "pending", "accepted", "dismissed"

4. **AmbientSession(BaseModel):**
   - `session_id: str` -- uuid4 hex[:8]
   - `state: str = Field(default="idle")` -- "idle", "recording", "processing", "results"
   - `transcript: EncounterTranscript | None = None`
   - `generated_note: StructuredNote | None = None`
   - `pipeline_result_json: dict | None = None` -- serialized PipelineResult (dict, not the model itself, for JSON compatibility in session state)
   - `disambiguation_items: list[DisambiguationItem] = Field(default_factory=list)`
   - `session_duration_seconds: float = 0.0`
   - `is_demo: bool = False` -- whether this was a pre-computed demo

5. **AmbientEncounterDemo(BaseModel):**
   - `encounter_id: str`
   - `encounter_label: str` -- display name for dropdown selector
   - `scenario_description: str` -- brief context shown in UI
   - `specialty: str = ""` -- e.g., "Primary Care", "Urgent Care"
   - `transcript: str` -- full conversation transcript text
   - `generated_note: str` -- pre-generated SOAP note text
   - `pipeline_result: dict` -- serialized PipelineResult JSON as dict
   - `disambiguation_items: list[dict] = Field(default_factory=list)` -- pre-computed disambiguation items
   - `session_duration_seconds: int = 180` -- simulated session duration

<task type="auto">
  <name>Task 2: Create ambient backend module with transcription, note generation, and pipeline integration</name>
  <files>cliniq/modules/m6_ambient.py</files>
  <action>
Create `cliniq/modules/m6_ambient.py` with three main functions. Follow the existing module pattern (see `m4_cdi.py` and `m5_explainability.py`): module-level docstring, logging, lazy imports for heavy deps.

**Function 1: `transcribe_audio(audio_path: str) -> EncounterTranscript`**
- Lazy-import `faster_whisper.WhisperModel` inside the function (not at module top) to avoid import-time model loading
- Use a module-level `_whisper_model` cache (same pattern as `_KG_CACHE` in m4_cdi.py): `_whisper_model: WhisperModel | None = None`
- Load model with `WhisperModel("small", device="cpu", compute_type="int8")` on first call
- Call `model.transcribe(audio_path, beam_size=5)` -- note: `segments` is a generator, must iterate
- Join all segment texts with spaces into a single transcript string
- Get duration from `info.duration` (returned by transcribe)
- Return `EncounterTranscript(raw_text=transcript, duration_seconds=duration)`
- Log the transcript length and duration

**Function 2: `generate_soap_note(transcript_text: str) -> StructuredNote`**
- Import `ModelManager` from `cliniq.model_manager`
- Get `(model, tokenizer)` from `ModelManager().get_reasoning_llm()`
- Use this prompt (include a brief few-shot example for quality):
```
You are a medical scribe. Convert this doctor-patient conversation transcript into a structured clinical note.

Transcript:
{transcript_text}

Write the clinical note with these sections:
- Chief Complaint (CC)
- History of Present Illness (HPI)
- Assessment
- Plan

Return ONLY the clinical note. Do not add commentary.

Clinical Note:
```
- Generate with `max_new_tokens=512, temperature=0.3, do_sample=True, pad_token_id=tokenizer.eos_token_id`
- Extract generated text after the prompt (same pattern as m3_rag_coding.py)
- Parse the generated text into StructuredNote fields by looking for section headers (CC/Chief Complaint, HPI/History, Assessment, Plan). Use simple string splitting -- if a section header is not found, leave that field empty. Always set `full_text` to the complete generated note text.
- If generation fails (exception or empty output), return a StructuredNote with `full_text=transcript_text` (fallback: use raw transcript as the note, which still works with run_pipeline_audited)
- Log success/failure

**Function 3: `run_ambient_pipeline(transcript_text: str, use_llm_queries: bool = False) -> tuple[StructuredNote, PipelineResult, list[DisambiguationItem]]`**
- Call `generate_soap_note(transcript_text)` to get the structured note
- Call `run_pipeline_audited(note.full_text, use_llm_queries=use_llm_queries)` from `cliniq.pipeline`
- Build disambiguation items from the CDI report:
  - For each `documentation_gap` in `result.cdi_report`: create `DisambiguationItem(item_id=uuid4().hex[:8], category="gap", title=f"Documentation Gap: {gap.code}", description=gap.physician_query, suggested_action=f"Clarify {gap.missing_qualifier} for {gap.code}", source_code=gap.code, confidence=gap.confidence)`
  - For each `missed_diagnosis`: create `DisambiguationItem(item_id=uuid4().hex[:8], category="missed_diagnosis", title=f"Potential Missed Dx: {md.suggested_code}", description=md.description, suggested_action=f"Consider documenting {md.suggested_code} ({md.description})", source_code=md.suggested_code, confidence=md.co_occurrence_weight)`
  - For each `code_conflict`: create `DisambiguationItem(item_id=uuid4().hex[:8], category="conflict", title=f"Code Conflict: {cc.code_a} vs {cc.code_b}", description=cc.conflict_reason, suggested_action=cc.recommendation, source_code=f"{cc.code_a},{cc.code_b}")`
- If `result.cdi_report` is None, return empty disambiguation list
- Return `(note, result, disambiguation_items)`
  </action>

<task type="auto">
  <name>Task 1: Clinical NER pipeline with entity mapping, negation detection, and qualifier capture</name>
  <files>cliniq/modules/m2_nlu.py</files>
  <action>
Build the complete NLU pipeline module with these functions:

**map_entity_type(model_label: str) -> str:**
- Use ENTITY_TYPE_MAP from cliniq.models.entities
- Return mapped category or "other" for unmapped labels
- Handle both bare labels ("Disease_disorder") and BIO-prefixed labels ("B-Disease_disorder") by stripping B-/I- prefix

**extract_raw_entities(text: str) -> list[dict]:**
- Get NER pipeline from ModelManager().get_ner_pipeline()
- Run pipeline on text: results = ner_pipe(text)
- The pipeline with aggregation_strategy="simple" already merges subword tokens
- Return raw results (list of dicts with keys: entity_group, score, word, start, end)

**detect_negation(text: str, entities: list[ClinicalEntity]) -> list[ClinicalEntity]:**
- Load spaCy model: nlp = spacy.load("en_core_web_sm")
- Add negex pipe: nlp.add_pipe("negex") — configure with termination terms like "but", "however", "although" to limit negation scope
- Create spaCy Doc from text
- For each entity, check if its text span falls within a negated scope
- Implementation approach: Create custom spaCy entities from our ClinicalEntity spans, run negex, read back ._.negex attribute
- Alternative simpler approach: For each entity, find its position in text. Look backward up to 6 tokens for negation triggers ("no", "not", "without", "denies", "denied", "no evidence of", "ruled out", "negative for"). If found and no termination term ("but", "however", "although", "except") intervenes, mark negated=True.
- Use the simpler approach if negspacy integration proves complex. The key requirement is accuracy >= 0.85 on negation cases.
- Return updated entity list with negated flags set

**capture_qualifiers(text: str, entities: list[ClinicalEntity]) -> list[ClinicalEntity]:**
- For entities of type "qualifier" (Severity, Detailed_description, etc.), find the nearest diagnosis/procedure entity within a 50-character window
- Attach the qualifier text to that entity's qualifiers list
- Remove standalone qualifier entities from the main list (they're now attributes of other entities)
- Example: "severe chest pain" -> ClinicalEntity(text="chest pain", entity_type="diagnosis", qualifiers=["severe"])

**extract_entities(text: str) -> NLUResult:**
- Main orchestrator function:
  1. Record start time
  2. Run extract_raw_entities(text)
  3. Map each raw entity to ClinicalEntity using map_entity_type
  4. Filter out entities with confidence < config.CONFIDENCE_THRESHOLD (0.80) -- but keep qualifier entities regardless of confidence
  5. Run detect_negation(text, entities)
  6. Run capture_qualifiers(text, entities)
  7. Record end time, compute processing_time_ms
  8. Return NLUResult(entities=entities, processing_time_ms=elapsed)

**process_document(doc: ClinicalDocument) -> NLUResult:**
- Convenience wrapper: extract_entities(doc.raw_narrative)
- Validates doc.raw_narrative is non-empty, raises ValueError if empty

Import time module for processing_time_ms. Use logging for debug output.

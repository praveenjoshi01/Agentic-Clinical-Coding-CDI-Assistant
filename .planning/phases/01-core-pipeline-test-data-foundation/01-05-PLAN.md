<task type="auto">
  <name>Task 1: RAG coding pipeline — retrieval, reranking, LLM reasoning, and sequencing</name>
  <files>cliniq/modules/m3_rag_coding.py</files>
  <action>
Build the complete RAG coding module with these functions:

**build_coding_query(entity: ClinicalEntity, context_window: str = "") -> str:**
- Construct search query from entity: "{entity.text}"
- If entity has qualifiers, append them: "{entity.text} {' '.join(entity.qualifiers)}"
- If context_window provided, append: " in context of {context_window}"
- This query goes to FAISS retrieval (BGE prefix is added by retriever, not here)

**retrieve_and_rerank(query: str, retriever: FAISSRetriever, reranker: CrossEncoderReranker) -> list[dict]:**
- Step 1: retriever.retrieve(query, top_k=config.RETRIEVAL_TOP_K) -> 20 candidates
- Step 2: reranker.rerank(query, candidates, top_k=config.RERANK_TOP_K) -> 5 candidates
- Return reranked candidates with both retrieval_score and rerank_score preserved

**reason_with_llm(entity: ClinicalEntity, candidates: list[dict], clinical_context: str) -> dict:**
- Get (model, tokenizer) from ModelManager().get_reasoning_llm()
- Build prompt:
  ```
  You are a clinical coding expert. Given a clinical finding and candidate ICD-10 codes, select the most specific appropriate code.

  Clinical finding: {entity.text}
  Qualifiers: {entity.qualifiers}
  Clinical context: {clinical_context[:500]}

  Candidate ICD-10 codes:
  {format each candidate as: "- {code}: {description} (score: {rerank_score:.2f})"}

  Return a JSON object with these exact fields:
  - "selected_code": the ICD-10 code string
  - "description": the code description
  - "confidence": your confidence 0.0-1.0
  - "reasoning": one sentence explaining why this code is most appropriate
  - "needs_specificity": true if a more specific code might exist but documentation is insufficient
  - "alternatives": [{"code": "X", "description": "Y", "reason": "Z"}] for top 2 alternatives

  Return ONLY the JSON object, no other text.
  ```
- Try to use outlines for constrained generation (import outlines, create JSON schema from CodeSuggestion-like structure, generate with constraints). If outlines integration fails, fall back to:
  1. Generate with model.generate(max_new_tokens=512, temperature=0.1)
  2. Extract JSON from response (find first { to last })
  3. Parse with json.loads
  4. Validate fields exist
  5. If parse fails, retry up to 2 more times with simplified prompt
- Return parsed dict with fields: selected_code, description, confidence, reasoning, needs_specificity, alternatives

**build_code_suggestion(entity: ClinicalEntity, llm_result: dict, candidates: list[dict]) -> CodeSuggestion:**
- Map llm_result fields to CodeSuggestion Pydantic model
- evidence_text = entity.text (the clinical text that triggered this code)
- If LLM confidence and rerank_score both available, use weighted average: 0.6*llm_confidence + 0.4*top_rerank_score
- Validate with Pydantic

**sequence_codes(suggestions: list[CodeSuggestion]) -> CodingResult:**
- Sequencing logic (simplified for POC, per research pitfall 7):
  1. Principal diagnosis: highest confidence suggestion that is a diagnosis (not procedure)
  2. Secondary codes (comorbidities): remaining diagnosis codes sorted by confidence
  3. Complication codes: any code where entity had qualifiers like "complication", "secondary to", "due to"
- Build sequencing_rationale string explaining the ordering
- Build retrieval_stats dict: {"total_entities_coded": N, "avg_confidence": X, "codes_needing_specificity": M}
- Return CodingResult

**code_entities(nlu_result: NLUResult, clinical_context: str = "") -> CodingResult:**
- Main orchestrator:
  1. Initialize FAISSRetriever and CrossEncoderReranker (ensure index is built)
  2. Filter entities: only process non-negated diagnoses and procedures (skip medications, anatomical_sites, negated entities)
  3. For each eligible entity:
     a. Build query with build_coding_query
     b. Retrieve and rerank with retrieve_and_rerank
     c. Reason with LLM via reason_with_llm
     d. Build CodeSuggestion via build_code_suggestion
  4. Sequence all suggestions via sequence_codes
  5. Return CodingResult
- Handle edge cases: empty NLU result (return CodingResult with empty fields), no FAISS results, LLM generation failure (use top reranked candidate as fallback)

Add logging throughout for debugging. Record timing for retrieval_stats.

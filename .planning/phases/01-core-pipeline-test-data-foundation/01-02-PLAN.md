**cliniq/data/icd10/icd10_codes.json:**
Create a curated JSON file with 200-500 ICD-10-CM codes commonly encountered in outpatient/professional fee coding. Each entry: {"code": "E11.9", "description": "Type 2 diabetes mellitus without complications", "chapter": "E00-E89"}. Include codes from these chapters at minimum:
- Endocrine (E00-E89): diabetes variants (E10.*, E11.*), thyroid, obesity
- Circulatory (I00-I99): hypertension (I10-I16), heart failure (I50.*), coronary artery (I25.*)
- Respiratory (J00-J99): pneumonia (J18.*), COPD (J44.*), asthma (J45.*)
- Musculoskeletal (M00-M99): back pain (M54.*), osteoarthritis (M17.*)
- Genitourinary (N00-N99): CKD stages (N18.*), UTI (N39.0)
- Mental (F00-F99): depression (F32.*), anxiety (F41.*)
- Neoplasms (C00-D49): common cancers
- Injury (S00-T88): fractures, burns
- Signs/symptoms (R00-R99): chest pain (R07.*), cough (R05), dyspnea (R06.0)

This is a DEMO dataset. Do not attempt to download the full CMS file. 200-500 carefully chosen codes with accurate descriptions is sufficient for the POC. Ensure all codes and descriptions are medically accurate.

**cliniq/rag/icd10_loader.py:**
- `load_icd10_codes(filepath: str | Path = None) -> list[dict]`: Loads from icd10_codes.json (default path from config.ICD10_DIR). Returns list of {"code": str, "description": str, "chapter": str}. Validates file exists, returns at least 100 codes.
- `get_code_by_id(codes: list[dict], code_id: str) -> dict | None`: Lookup helper.
- `get_codes_by_chapter(codes: list[dict], chapter_prefix: str) -> list[dict]`: Filter by chapter.

**cliniq/rag/build_index.py:**
- `build_faiss_index(codes: list[dict] = None, output_dir: Path = None) -> tuple[faiss.Index, list[dict]]`:
  1. Load codes via icd10_loader if not provided
  2. Extract descriptions list
  3. Load SentenceTransformer("BAAI/bge-small-en-v1.5") -- use ModelManager.get_embedder()
  4. Encode descriptions with normalize_embeddings=True, batch_size=256 (NO query prefix for documents)
  5. Create faiss.IndexFlatIP (inner product = cosine similarity with normalized vectors)
  6. Add embeddings to index
  7. Save index to output_dir/icd10.faiss using faiss.write_index
  8. Save code metadata to output_dir/icd10_metadata.json
  9. Return (index, codes)
- `load_faiss_index(index_dir: Path = None) -> tuple[faiss.Index, list[dict]]`: Load pre-built index and metadata from disk. Default path from config.INDEX_DIR.
  </action>

**cliniq/rag/retriever.py:**
- `class FAISSRetriever`:
  - `__init__(self, index_dir: Path = None)`: Load pre-built FAISS index and metadata via load_faiss_index. Store embedder reference (lazy via ModelManager).
  - `retrieve(self, query: str, top_k: int = 20) -> list[dict]`:
    1. Prepend BGE_QUERY_PREFIX to query: `"Represent this sentence: " + query`
    2. Encode query with embedder (normalize_embeddings=True)
    3. Search FAISS index: distances, indices = index.search(query_vec.reshape(1, -1), top_k)
    4. Build results: list of {"code": str, "description": str, "score": float, "rank": int}
    5. Return sorted by score descending
  - `ensure_index_built(self)`: Check if index files exist, build if missing.

**cliniq/rag/reranker.py:**
- `class CrossEncoderReranker`:
  - `__init__(self)`: Get cross-encoder from ModelManager (lazy).
  - `rerank(self, query: str, candidates: list[dict], top_k: int = 5) -> list[dict]`:
    1. Create pairs: [(query, c["description"]) for c in candidates]
    2. Get scores from cross_encoder.predict(pairs)
    3. Add "rerank_score" to each candidate dict
    4. Sort by rerank_score descending
    5. Return top_k candidates
    6. Preserve original "score" as "retrieval_score" for comparison

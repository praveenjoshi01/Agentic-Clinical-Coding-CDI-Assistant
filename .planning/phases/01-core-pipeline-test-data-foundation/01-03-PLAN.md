<task type="auto">
  <name>Task 1: Multi-modal ingestion module with FHIR, text, and image parsers</name>
  <files>cliniq/modules/m1_ingest.py</files>
  <action>
Create the ingestion module with four functions and one router.

**detect_modality(input_data) -> Literal["fhir", "text", "image"]:**
- If input_data is str/Path ending with .png, .jpg, .jpeg, .bmp, .tiff -> "image"
- If input_data is dict with "resourceType" key -> "fhir"
- If input_data is str, try json.loads: if dict with "resourceType" -> "fhir", else -> "text"
- If str and not valid JSON -> "text"
- Raise ValueError for unrecognized types

**parse_fhir(fhir_data: dict | str) -> ClinicalDocument:**
- If str, json.loads it first
- Use `from fhir.resources.r4b.bundle import Bundle` (NOT default R5 imports -- per research pitfall 2)
- Also import: Condition, Procedure, Encounter from fhir.resources.r4b
- Validate with Bundle.model_validate(fhir_data)
- Iterate bundle.entry: for each entry.resource, check resource_type
  - "Condition": extract code.text, clinicalStatus, coding systems -> append to narrative_parts and structured_facts
  - "Procedure": extract code.text, status -> append to narrative_parts and structured_facts
  - "Encounter": extract class, period, type -> structured_facts
  - "DocumentReference": if content exists, extract text attachment -> raw narrative
- Extract patient_id from Patient resource reference or generate UUID
- Extract encounter_id from Encounter resource or generate UUID
- Return ClinicalDocument(metadata=DocumentMetadata(patient_id, encounter_id, source_type="fhir"), raw_narrative=joined narrative_parts, structured_facts=facts, modality_confidence=1.0, extraction_trace="Parsed from FHIR R4 Bundle")

**parse_text(text: str) -> ClinicalDocument:**
- Simple wrapper: return ClinicalDocument with source_type="text", raw_narrative=text.strip(), modality_confidence=1.0, extraction_trace="Direct text input", generate UUID for patient/encounter IDs.

**parse_image(image_path: str | Path) -> ClinicalDocument:**
- Validate file exists, is readable
- Use ModelManager().get_multimodal() to get (model, processor)
- Load image with PIL.Image.open
- Build chat message with role="user", content=[{"type": "image"}, {"type": "text", "text": "Extract all clinical text from this medical document. Include all diagnoses, procedures, medications, and clinical findings exactly as written."}]
- Apply processor.apply_chat_template with add_generation_prompt=True
- Process with processor(text=prompt, images=[image], return_tensors="pt")
- Generate with model.generate(**inputs, max_new_tokens=1024)
- Decode with processor.batch_decode(generated_ids, skip_special_tokens=True)[0]
- Clean extracted text: strip chat template artifacts, keep only the assistant's response
- Confidence heuristic: clamp between 0.4 and 0.85 based on output length (len(text)/500)
- Return ClinicalDocument with source_type="image", raw_narrative=extracted_text, modality_confidence=confidence

**ingest(input_data) -> ClinicalDocument:**
- Router: detect_modality -> dispatch to correct parser
- Validate result with ClinicalDocument model validation (Pydantic will do this automatically)
- Return the ClinicalDocument

Add appropriate error handling: wrap FHIR parsing in try/except for ValidationError, wrap image parsing for FileNotFoundError and model errors. Log errors with extraction_trace.
  </action>

  </verify>

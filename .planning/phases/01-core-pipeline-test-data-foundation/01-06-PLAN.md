**cliniq/pipeline.py:**

Create the pipeline orchestrator that chains all modules:

```python
class PipelineResult(BaseModel):
    document: ClinicalDocument
    nlu_result: NLUResult
    coding_result: CodingResult
    processing_time_ms: float
    errors: list[str] = Field(default_factory=list)
```

**run_pipeline(input_data, skip_coding: bool = False) -> PipelineResult:**
1. Record start time
2. Stage 1 — Ingestion: doc = ingest(input_data). If fails, return PipelineResult with error.
3. Stage 2 — NER: nlu_result = extract_entities(doc.raw_narrative). If fails or returns 0 entities, log warning but continue.
4. Stage 3 — RAG Coding (unless skip_coding=True): coding_result = code_entities(nlu_result, clinical_context=doc.raw_narrative). If fails, return partial result with error.
5. Record end time
6. Return PipelineResult with all results and total processing_time_ms

Add try/except around each stage. Capture errors in PipelineResult.errors list. Each stage should proceed even if prior stage partially failed (e.g., if NER finds 0 entities, RAG coding returns empty CodingResult).

**run_pipeline_batch(inputs: list, skip_coding: bool = False) -> list[PipelineResult]:**
- Process multiple inputs sequentially
- Return list of PipelineResult

**cliniq/tests/test_pipeline.py:**
1. test_pipeline_with_text: run_pipeline("Patient has type 2 diabetes and hypertension.") returns PipelineResult with non-empty document, nlu_result. Mark @pytest.mark.slow.
2. test_pipeline_with_fhir: run_pipeline(sample_fhir_bundle_dict) returns PipelineResult with source_type="fhir". Mark @pytest.mark.slow.
3. test_pipeline_error_handling: run_pipeline("") should not crash, should have error in errors list or handle gracefully.
4. test_pipeline_result_schema: Verify PipelineResult validates with Pydantic (non-model test).
5. test_pipeline_skip_coding: run_pipeline with skip_coding=True returns result without coding_result populated.

**For each case, the gold standard entry (GoldStandardCase) must include:**
- case_id: "case_001" through "case_020"
- source_type: "fhir", "text", or "image"
- input_data: file path relative to project root (e.g., "cliniq/data/gold_standard/fhir_bundles/case_001.json")
- expected_entities: list of GoldStandardEntity with text, entity_type (using pipeline categories: diagnosis, procedure, medication, anatomical_site, qualifier), start_char, end_char, negated flag, qualifiers list. Include at least 3-5 entities per case.
- expected_icd10_codes: list of correct ICD-10-CM codes (e.g., ["E11.40", "E11.311", "I10"]). Use real, specific codes from the ICD10 dataset.
- expected_principal_dx: the primary diagnosis code
- expected_comorbidities: secondary diagnosis codes
- expected_complications: complication codes (if applicable)
- negation_test_cases: at least 2 per case with {"text": "no pneumonia", "entity": "pneumonia", "negated": true}. Include both negated and affirmed examples.
- cdi_gap_annotations: hints for Phase 2 CDI gaps (e.g., {"code": "N18.3", "missing_qualifier": "specific GFR range", "query_hint": "What is the patient's current eGFR?"}). Include 1-2 per case.
- kg_qualification_rules: hints for Phase 2 KG rules (e.g., {"code": "E11.40", "requires": ["laterality", "severity"]}). Include 1-2 per case.
- notes: brief description of what this case tests

**FHIR bundle format (for cases 1-3, 20):**
- Valid FHIR R4 Bundle with resourceType="Bundle", type="collection"
- Include Patient resource with name, gender, birthDate
- Include Encounter resource with class, type, period
- Include 2-3 Condition resources with code (text + ICD-10 coding), clinicalStatus
- Include 1-2 Procedure resources if relevant, with code, status
- Use proper FHIR R4 structure that will validate with fhir.resources.r4b

**Text note format (for cases 4-14):**
- 150-300 word clinical notes in standard HPI/exam/assessment format
- Include explicit negations (e.g., "denies chest pain", "no fever")
- Include severity qualifiers (e.g., "severe", "stage 3", "acute", "bilateral")
- Include medication mentions with dosages

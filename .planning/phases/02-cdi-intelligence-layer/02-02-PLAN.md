Create KG querier module with three query functions. The querier takes a frozen nx.DiGraph and case-specific data, returns structured results. It NEVER calls add_node/add_edge — read-only queries only.

**cliniq/knowledge_graph/querier.py:**

`find_documentation_gaps(G: nx.DiGraph, case_codes: list[str], entity_qualifiers: dict[str, list[str]]) -> list[dict]` (CDI-01):
- For each code in case_codes, check if KG has the node.
- Look at REQUIRES_QUALIFIER out-edges from that code.
- Compare required qualifiers against entity_qualifiers[code] (documented qualifiers from NER).
- If a required qualifier is NOT in the documented list, add a gap: {"code": code, "missing_qualifier": qualifier, "description": G.nodes[code].get("description", "")}.
- Return list of all gaps found.
- Handle missing codes gracefully (skip codes not in KG).

`find_code_conflicts(G: nx.DiGraph, case_codes: list[str]) -> list[dict]` (CDI-04):
- Check all pairs of case_codes for CONFLICTS_WITH edges (check both directions since bidirectional).
- For each conflict found: {"code_a": code_a, "code_b": code_b, "reason": edge.get("reason", "Excludes1 conflict")}.
- Deduplicate: don't report (A,B) AND (B,A) — use a set of frozensets to track seen pairs.
- Return list of conflicts.

`find_missed_diagnoses(G: nx.DiGraph, case_codes: list[str], max_suggestions: int = 5) -> list[dict]` (CDI-03):
- For each code in case_codes, find COMMONLY_CO_CODED out-edges.
- If the neighbor code is NOT already in case_codes, it's a potential missed diagnosis.
- Collect: {"suggested_code": neighbor, "description": G.nodes.get(neighbor, {}).get("description", ""), "co_coded_with": code, "weight": edge.get("weight", 0.0)}.
- Sort by weight descending, deduplicate by suggested_code (keep highest weight), limit to max_suggestions.
- Return sorted list.

Update `cliniq/knowledge_graph/__init__.py` to also re-export: find_documentation_gaps, find_code_conflicts, find_missed_diagnoses from querier.

  <action>
Extend the existing pipeline.py with CDI analysis stage and audit trail. IMPORTANT: Do NOT break existing run_pipeline function or PipelineResult schema. Add new fields as Optional with default None.

**Extend PipelineResult:**
- Add `cdi_report: Optional[CDIReport] = None` — CDI analysis output (populated by run_pipeline_audited)
- Add `audit_trail: Optional[AuditTrail] = None` — audit trail (populated by run_pipeline_audited)
- Import CDIReport from cliniq.models.cdi and AuditTrail from cliniq.models.audit.

**Add run_pipeline_audited function:**

`run_pipeline_audited(input_data, skip_coding=False, skip_cdi=False, use_llm_queries=True) -> PipelineResult:`

This wraps each existing pipeline stage with audit trail capture:

1. **Create AuditTrailBuilder** with auto-generated case_id (uuid4 short).

2. **Stage 1 — Ingestion (instrumented):**
   - Record time before/after ingest() call.
   - Call ingest(input_data) as before.
   - Record StageTrace: stage="ingestion", processing_time_ms, input_summary=str(type(input_data)), output_summary=f"{source_type}, confidence={modality_confidence}".
   - Error handling same as existing run_pipeline.

3. **Stage 2 — NER (instrumented):**
   - Record time before/after extract_entities().
   - Call extract_entities(document.raw_narrative) as before.
   - Record StageTrace: stage="ner", processing_time_ms, input_summary=f"{len(narrative)} chars", output_summary=f"{entity_count} entities".
   - Include entity details in trace details dict.

4. **Stage 3 — RAG Coding (instrumented):**
   - Call code_entities(nlu_result, clinical_context) as before.
   - Record StageTrace: stage="rag", processing_time_ms (from retrieval_stats), input_summary=f"{entity_count} entities", output_summary=f"principal={principal_code}, {n_secondary} secondary".
   - Use link_evidence_spans(coding_result, clinical_text) from m5_explainability to populate evidence_spans in the audit trail.

5. **Stage 4 — CDI Analysis (new, unless skip_cdi=True):**
   - Record time before/after run_cdi_analysis().
   - Call run_cdi_analysis(nlu_result, coding_result, clinical_text, use_llm_queries).
   - Record StageTrace: stage="cdi", processing_time_ms, input_summary=f"{n_codes} codes", output_summary=f"{gap_count} gaps, {conflict_count} conflicts, {len(missed)} missed dx".
   - Store CoT traces from CDI gaps in the stage trace's cot_traces list.

6. **Assemble PipelineResult** with all fields including cdi_report and audit_trail.

**Keep existing run_pipeline UNCHANGED.** The new run_pipeline_audited is a separate function. Both return PipelineResult, but run_pipeline returns it with cdi_report=None and audit_trail=None (backward compatible).

**Add run_pipeline_audited_batch** analogous to run_pipeline_batch but using run_pipeline_audited.

Import new dependencies at top: from cliniq.modules.m4_cdi import run_cdi_analysis, from cliniq.modules.m5_explainability import AuditTrailBuilder, link_evidence_spans.

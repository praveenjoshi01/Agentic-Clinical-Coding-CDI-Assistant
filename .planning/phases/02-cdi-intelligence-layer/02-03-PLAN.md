**cliniq/modules/m5_explainability.py:**

**class AuditTrailBuilder:**
Convenience wrapper around AuditTrail that provides stage-recording methods.

- `__init__(self, case_id: str)`: Creates an AuditTrail instance with the given case_id.
- `record_stage(self, stage: str, processing_time_ms: float, input_summary: str, output_summary: str, details: dict = None, cot_traces: list[str] = None, retrieval_logs: list[RetrievalLog] = None)`: Creates a StageTrace and appends it to the trail. Handles None defaults.
- `add_evidence(self, code: str, text_span: str)`: Delegates to AuditTrail.add_evidence(). For EXPL-02.
- `get_trail(self) -> AuditTrail`: Returns the accumulated audit trail.
- Property `stage_count -> int`: Number of stages recorded.
- Property `has_all_stages -> bool`: Returns True if trail has at least one trace for each of the 4 core stages: "ingestion", "ner", "rag", "cdi".

**capture_cot_and_json(raw_llm_response: str) -> tuple[str, str]:**
Shared utility for EXPL-03. Takes the full decoded Qwen response text:
1. Store the full raw_llm_response as cot_trace.
2. Find first '{' and last '}' to extract JSON substring.
3. Return (cot_trace, json_str). If no JSON found, return (cot_trace, "").
This function is used by both Phase 1 RAG reasoning (via pipeline wrapper) and Phase 2 CDI query generation.

**link_evidence_spans(coding_result: CodingResult, clinical_text: str) -> dict[str, list[str]]:**
For EXPL-02. Maps each code in the CodingResult to supporting text spans:
1. Collect all CodeSuggestions from principal + secondary + complication.
2. For each suggestion, use evidence_text field as the primary span.
3. Additionally, search clinical_text for the entity text (case-insensitive substring match) and capture a window of +/- 50 characters around each match as contextual evidence.
4. Return dict mapping icd10_code -> list of text spans (deduplicated).

**build_retrieval_log(query: str, top_k_results: list[dict], reranked_results: list[dict], selected_code: str, selected_confidence: float) -> RetrievalLog:**
For EXPL-04. Creates a RetrievalLog Pydantic instance from retrieval pipeline intermediate results. Used by the pipeline wrapper to capture the full retrieval chain.

Import from cliniq.models: AuditTrail, StageTrace, RetrievalLog, CodingResult, CodeSuggestion.

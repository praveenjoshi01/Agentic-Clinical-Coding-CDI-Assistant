Create the CDI agent module that orchestrates KG queries and LLM-based physician query generation.

**cliniq/modules/m4_cdi.py:**

**generate_physician_query(gap: dict, clinical_context: str) -> tuple[str, str]** (CDI-02):
- Gets Qwen model + tokenizer from ModelManager().get_reasoning_llm().
- Builds prompt with: ICD-10 code, code description, missing qualifier, clinical context (first 300 chars), and 2 few-shot examples of good physician queries.
- Few-shot examples:
  - Gap: E11.40 missing "laterality" -> "Dr. Smith, the documentation notes diabetic neuropathy. Could you please specify whether the neuropathy affects the right, left, or bilateral lower extremities?"
  - Gap: I50.9 missing "type" -> "The assessment indicates heart failure. Could you clarify whether this is systolic (HFrEF), diastolic (HFpEF), or combined heart failure?"
- Generates with max_new_tokens=256, temperature=0.1, do_sample=False.
- Uses capture_cot_and_json from m5_explainability to capture raw CoT trace (EXPL-03).
- Extracts "query" field from JSON response.
- **Template fallback**: If JSON parsing fails after extraction attempt, generate a template-based query: "Can you please clarify the {missing_qualifier} for the documented {description}? This information is needed to assign the most specific ICD-10 code ({code})."
- Returns (physician_query_text, raw_cot_trace).

**_extract_entity_qualifiers(nlu_result: NLUResult, coding_result: CodingResult) -> dict[str, list[str]]:**
Helper to build the entity_qualifiers dict needed by find_documentation_gaps:
- For each CodeSuggestion in coding_result (principal + secondary + complication):
  - Find matching entities in nlu_result by comparing evidence_text to entity.text.
  - Collect entity.qualifiers for that code.
- Return dict mapping icd10_code -> list of documented qualifiers.

**_find_evidence_for_code(code: str, nlu_result: NLUResult, clinical_text: str) -> str:**
Helper to find supporting text span for a suggested code (EXPL-02):
- Search nlu_result.entities for entities whose qualifiers or text relate to the code description.
- Fall back to searching clinical_text for relevant keywords.
- Return the best matching text span (up to 200 chars).

**calculate_completeness_score(gaps: list, conflicts: list, total_codes: int) -> float** (CDI-05):
- Completeness = 1.0 - (gap_penalty + conflict_penalty).
- gap_penalty = len(gaps) * 0.1 (each gap reduces score by 10%).
- conflict_penalty = len(conflicts) * 0.15 (each conflict reduces score by 15%).
- Clamp to [0.0, 1.0].
- If total_codes == 0, return 1.0 (no codes to evaluate).

**run_cdi_analysis(nlu_result: NLUResult, coding_result: CodingResult, clinical_text: str, use_llm_queries: bool = True) -> CDIReport** (CDI-05 orchestrator):
Main orchestrator function:
1. Build the static KG via build_cdi_knowledge_graph() (cached at module level — build once, reuse).
2. Extract case codes from CodingResult (principal + secondary + complication icd10_codes).
3. Extract entity qualifiers via _extract_entity_qualifiers().
4. Call find_documentation_gaps(G, case_codes, entity_qualifiers) for CDI-01.
5. Call find_code_conflicts(G, case_codes) for CDI-04.
6. Call find_missed_diagnoses(G, case_codes) for CDI-03.
7. For each documentation gap, if use_llm_queries=True:
   - Call generate_physician_query(gap, clinical_text) for CDI-02.
   - Find evidence span via _find_evidence_for_code().
   - Create DocumentationGap Pydantic model with all fields including physician_query and cot_trace.
   If use_llm_queries=False: use template fallback directly (for fast testing).
8. For each missed diagnosis: create MissedDiagnosis model with evidence from _find_evidence_for_code().
9. For each conflict: create CodeConflict model with recommendation ("Review documentation to determine which code is clinically appropriate").
10. Calculate completeness_score.
11. Return CDIReport with all lists + completeness_score + processing_time_ms.

**Module-level KG caching:** `_KG_CACHE: nx.DiGraph | None = None` with `_get_kg()` function that builds on first call and caches.

Use logging throughout. Import from knowledge_graph, models, model_manager, m5_explainability.

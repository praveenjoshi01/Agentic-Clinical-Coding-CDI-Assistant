**cliniq/evaluation/llm_judge.py:**

**judge_query_relevance(physician_query: str, documentation_gap: str, clinical_context: str) -> dict** (CDI-06):
- Gets Qwen model + tokenizer from ModelManager.
- Builds judge prompt with explicit 5-point Likert rubric:
  ```
  Rate the relevance of this physician query for addressing the documentation gap.

  Documentation Gap: {documentation_gap}
  Clinical Context: {clinical_context[:300]}
  Physician Query: {physician_query}

  Scoring rubric:
  1 = Completely irrelevant — query does not address the documentation gap at all
  2 = Tangentially related — query mentions the condition but not the specific missing information
  3 = Somewhat relevant — query addresses the gap but is too vague or broad to be actionable
  4 = Relevant and specific — query clearly asks about the missing information with clinical precision
  5 = Highly relevant, specific, and actionable — query references the specific clinical finding, asks about the exact missing qualifier, and uses appropriate medical language

  Return ONLY a JSON: {"score": N, "reasoning": "brief explanation"}
  ```
- Generates with max_new_tokens=128, temperature=0.1.
- Uses capture_cot_and_json for parsing.
- Parses score from JSON, normalizes to 0-1 by dividing by 5.
- Returns {"score": normalized_score, "raw_score": int_score, "reasoning": str, "cot_trace": str}.
- Fallback: if JSON parsing fails, return {"score": 0.6, "raw_score": 3, "reasoning": "Parse failure, default score", "cot_trace": raw_response}.

**judge_cot_coherence(cot_trace: str, task_description: str) -> dict** (EXPL-05):
- Similar structure to query relevance judge.
- Rubric for coherence:
  ```
  Rate the coherence of this chain-of-thought reasoning trace.

  Task: {task_description}
  Chain-of-Thought: {cot_trace[:500]}

  Scoring rubric:
  1 = Incoherent — reasoning is garbled, contradictory, or unrelated to the task
  2 = Partially coherent — some relevant reasoning but with logical gaps or non-sequiturs
  3 = Coherent but shallow — reasoning follows a logical thread but lacks depth or clinical insight
  4 = Coherent and substantive — reasoning demonstrates clear clinical logic with appropriate steps
  5 = Highly coherent — reasoning is logically structured, clinically sound, and demonstrates expertise

  Return ONLY a JSON: {"score": N, "reasoning": "brief explanation"}
  ```
- Same generation/parsing/fallback pattern as query relevance.
- Returns {"score": normalized, "raw_score": int, "reasoning": str, "cot_trace": str}.

**evaluate_cdi_quality(cdi_reports: list[CDIReport], clinical_contexts: list[str]) -> dict** (aggregate CDI-06):
- For each CDIReport, score every DocumentationGap's physician_query via judge_query_relevance.
- Aggregate: mean_relevance_score, min_score, max_score, n_queries_scored, n_above_threshold (>= 0.80).
- Return summary dict with aggregate metrics + per-query details.

**evaluate_cot_quality(audit_trails: list[AuditTrail]) -> dict** (aggregate EXPL-05):
- For each AuditTrail, score every cot_trace in every StageTrace via judge_cot_coherence.
- Aggregate: mean_coherence_score, min_score, max_score, n_traces_scored, n_above_threshold (>= 0.82).
- Return summary dict with aggregate metrics + per-trace details.

Import from cliniq.model_manager, cliniq.models (CDIReport, AuditTrail), cliniq.modules.m5_explainability (capture_cot_and_json).

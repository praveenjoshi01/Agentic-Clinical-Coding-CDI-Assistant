<task type="auto">
  <name>Task 2: QA Bot page with chat interface and pre-seeded questions</name>
  <files>ui/pages/qa_bot.py</files>
  <action>
    Replace the stub qa_bot.py with the full implementation.

    **QA BOT DESIGN DECISION:**
    Since running Qwen2.5-1.5B for RAG requires model downloads and is slow on CPU, the QA Bot uses a two-tier approach:
    1. **Pre-seeded questions (primary):** Match user input against demo_questions.json using simple keyword/fuzzy matching. Returns pre-written, verified answers instantly.
    2. **LLM fallback (optional):** If the question doesn't match any pre-seeded question AND the user has enabled LLM mode, use Qwen via RAG. Default: pre-seeded only.

    **PAGE LAYOUT:**
    - st.title("QA Bot") with subtitle "Ask questions about the ClinIQ system"

    **SIDEBAR - Quick Questions:**
    - st.sidebar.markdown("### Interview Questions")
    - Load questions from ui/demo_data/demo_questions.json
    - For each question: st.sidebar.button(question_text, key=f"q_{i}")
    - On button click: set st.session_state["pending_question"] = question and st.rerun()

    **SIDEBAR - Settings:**
    - Toggle: "Use LLM for unknown questions" (default False)
    - Help text: "When enabled, uses Qwen2.5-1.5B to answer questions not in the pre-seeded list. Requires model download."

    **CHAT INTERFACE:**
    - Initialize st.session_state["messages"] if not exists: [{"role": "assistant", "content": "Welcome! I'm the ClinIQ QA Bot. Ask me about the pipeline architecture, models, evaluation metrics, or any aspect of the system. Try the pre-seeded questions in the sidebar for instant answers!"}]
    - Display all messages using st.chat_message(msg["role"]): st.markdown(msg["content"])
    - Accept input via st.chat_input("Ask about the pipeline, models, evaluation...")
    - Handle pending_question from sidebar buttons

    **ANSWER GENERATION:**
    - Load demo_questions.json into a list of {"question": str, "answer": str} dicts
    - For incoming question, find best match:
      * Normalize both strings (lowercase, strip)
      * Check each pre-seeded question for keyword overlap (split into words, compute Jaccard similarity)
      * If best match has Jaccard >= 0.3 (at least 30% word overlap): return the pre-seeded answer
      * Otherwise: if LLM mode enabled, try to generate (but for now, return a polite "I don't have specific information about that. Try one of the pre-seeded questions in the sidebar for verified answers about the ClinIQ system.")
    - Append user message and assistant response to st.session_state["messages"]

    **ANSWER RENDERING:**
    - Use st.chat_message("assistant") for bot responses
    - Show a small badge: "[Pre-seeded]" or "[Generated]" to indicate answer source
    - For pre-seeded answers: render immediately (no spinner)
    - For generated answers: show st.spinner("Thinking...")

    **IMPORTANT:**
    - Do NOT import ModelManager or any transformers modules at page load
    - Only import LLM-related code inside the LLM fallback code path
    - Chat history persists in session_state["messages"] across page navigations (UI-07)
    - Pre-seeded questions must cover the 7 questions defined in demo_questions.json (UI-06)
  </action>

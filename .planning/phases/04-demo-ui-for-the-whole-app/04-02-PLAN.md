  <action>
    Replace the stub pipeline_runner.py with the full implementation. The page has 3 sections: Input, Execution, Results.

    **INPUT SECTION:**
    - st.title("Pipeline Runner") with descriptive subtitle
    - Radio for input type: "Text Note", "FHIR Bundle (JSON)", "Scanned Image" (horizontal=True)
    - For Text Note: st.text_area with placeholder clinical note text, height=200
    - For FHIR Bundle: st.file_uploader accepting .json files
    - For Scanned Image: st.file_uploader accepting .png, .jpg, .jpeg
    - Toggle: "Use pre-computed results" (default True) with help text explaining instant vs live
    - Demo case selector: st.selectbox with options:
      * "(none)"
      * "Case 004: CKD + Hypertension (68M, text)" -> case_004
      * "Case 010: CHF + AFib (75M, text)" -> case_010
      * "Case 001: Diabetes + Neuropathy (FHIR)" -> case_001
    - "Run Pipeline" primary button, full width

    **EXECUTION SECTION (on button click):**
    - If use_precomputed is True AND a demo case is selected:
      * Load from ui/demo_data/precomputed/{case_id}.json
      * Deserialize with PipelineResult.model_validate_json(json_text)
      * If file doesn't exist, show st.warning("Pre-computed results not found. Run scripts/precompute_demo.py first.") and fall through to live execution
    - If live execution:
      * Call run_pipeline_with_status(input_data, use_llm_queries=False) from pipeline_status component
      * For file uploads: read file bytes, parse JSON for FHIR, save temp file for images
    - Store result in st.session_state["pipeline_result"] and st.session_state["active_case_id"]

    **RESULTS SECTION (if pipeline_result exists in session state):**
    Display results in tabbed sections using st.tabs: "Overview", "NER Entities", "ICD-10 Codes", "CDI Analysis"

    Tab 1 - Overview:
    - Metric row: Processing Time, Entity Count, Code Count, Completeness Score
    - Source type badge, patient ID, encounter ID from document metadata
    - Error list if any errors occurred

    Tab 2 - NER Entities:
    - Call render_ner_highlights(result.document.raw_narrative, result.nlu_result.entities) for inline annotation
    - Below: entity summary table with columns: Entity Text, Type, Negated, Confidence, Qualifiers
    - Color legend for entity types

    Tab 3 - ICD-10 Codes:
    - Call render_principal_diagnosis(result.coding_result.principal_diagnosis)
    - Call render_code_cards(result.coding_result.secondary_codes, "Secondary Codes")
    - Call render_code_cards(result.coding_result.complication_codes, "Complications")
    - Show sequencing rationale
    - Show retrieval stats if available

    Tab 4 - CDI Analysis:
    - If result.cdi_report is None: st.info("CDI analysis was not run for this case.")
    - Otherwise:
      * Completeness score gauge (st.metric with delta showing gap/conflict penalty)
      * Documentation Gaps: expandable cards with code, description, missing qualifier, physician query, evidence text
      * Missed Diagnoses: cards with suggested code, co-coded with, weight
      * Code Conflicts: warning cards with code_a vs code_b, reason, recommendation
    - Navigation hints: st.page_link to KG Viewer and Audit Trail pages

    **IMPORTANT PATTERNS:**
    - NEVER call run_pipeline_audited() outside the button click guard
    - Always check for session_state["pipeline_result"] before rendering results
    - Use @st.cache_data for loading pre-computed JSON
    - Handle the case where PipelineResult is already in session state (from a previous run) by showing results immediately

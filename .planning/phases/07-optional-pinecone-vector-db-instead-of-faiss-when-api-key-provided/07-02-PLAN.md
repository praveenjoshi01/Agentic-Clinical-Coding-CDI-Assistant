Create `scripts/populate_pinecone_index.py` following the same CLI pattern as `scripts/build_v2_index.py` (PROJECT_ROOT sys.path injection, argparse, error handling):

**Script flow:**
1. `PROJECT_ROOT = Path(__file__).resolve().parent.parent; sys.path.insert(0, str(PROJECT_ROOT))`
2. Accept `--openai-api-key` and `--pinecone-api-key` as CLI args (fall back to `OPENAI_API_KEY` and `PINECONE_API_KEY` env vars)
3. Accept `--check` flag to just check if index exists and print vector count
4. Accept `--delete` flag to delete the index (for re-creation)

**populate_index(openai_key, pinecone_key) function:**
1. Configure both clients:
   ```python
   from cliniq_v2.api_client import OpenAIClient
   from cliniq_v2.pinecone_client import PineconeClient
   OpenAIClient().configure(openai_key)
   PineconeClient().configure(pinecone_key)
   ```
2. Validate both keys (OpenAIClient().validate_key(), PineconeClient().validate_key())
3. Import from pinecone: `from pinecone import ServerlessSpec`
4. Import from config: `PINECONE_INDEX_NAME, PINECONE_NAMESPACE, EMBEDDING_DIMENSIONS`
5. Load ICD-10 codes: `from cliniq.rag.icd10_loader import load_icd10_codes; codes = load_icd10_codes()`
6. Create index if not exists:
   ```python
   pc = PineconeClient().client
   if not pc.has_index(name=PINECONE_INDEX_NAME):
       pc.create_index(
           name=PINECONE_INDEX_NAME,
           dimension=EMBEDDING_DIMENSIONS,
           metric="cosine",
           spec=ServerlessSpec(cloud="aws", region="us-east-1"),
       )
       # Wait for ready
       import time
       while not pc.describe_index(PINECONE_INDEX_NAME).status["ready"]:
           time.sleep(1)
       print(f"Index '{PINECONE_INDEX_NAME}' created.")
   else:
       print(f"Index '{PINECONE_INDEX_NAME}' already exists.")
   ```
7. Embed descriptions with OpenAI in batches of 200 (not 2048 -- keep batch size manageable for 265 codes, a single batch is fine but batch for safety):
   ```python
   client = OpenAIClient().client
   descriptions = [c["description"] for c in codes]
   # Single API call is fine for 265 descriptions
   response = client.embeddings.create(model="text-embedding-3-small", input=descriptions)
   embeddings = [item.embedding for item in response.data]
   ```
8. Prepare vectors for upsert:
   ```python
   vectors = []
   for code_entry, embedding in zip(codes, embeddings):
       vectors.append({
           "id": code_entry["code"],
           "values": embedding,
           "metadata": {
               "code": code_entry["code"],
               "description": code_entry["description"],
               "chapter": code_entry.get("chapter", ""),
           },
       })
   ```
9. Upsert to Pinecone in batches of 200:
   ```python
   index = pc.Index(name=PINECONE_INDEX_NAME)
   for i in range(0, len(vectors), 200):
       batch = vectors[i : i + 200]
       index.upsert(vectors=batch, namespace=PINECONE_NAMESPACE)
   ```
10. Wait briefly then verify: `stats = index.describe_index_stats(); print(f"Vectors: {stats.total_vector_count}")`
11. Print success summary with timing.

**check_index(pinecone_key) function:**
- Configure PineconeClient, check has_index(), if exists get stats and print vector count per namespace.

**delete_index(pinecone_key) function:**
- Configure PineconeClient, delete index if exists, print confirmation.

**main() function:**
- argparse with `--openai-api-key`, `--pinecone-api-key`, `--check`, `--delete` flags
- Help epilog with usage examples (same style as build_v2_index.py)
- KeyboardInterrupt handler

**Error handling:**
- Specific messages for: invalid API keys, missing ICD-10 data (FileNotFoundError), rate limits, index creation failures
- Same pattern as build_v2_index.py (try/except with sys.exit(1))

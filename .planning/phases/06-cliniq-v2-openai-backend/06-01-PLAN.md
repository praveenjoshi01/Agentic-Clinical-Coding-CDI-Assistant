  - `_instance: Optional["OpenAIClient"] = None` (class var)

  - `configure(api_key: str)` initialises `self._client = OpenAI(api_key=api_key)`
  - `client` property raises RuntimeError if not configured
  - `validate_key() -> bool` calls `self.client.models.list()` in try/except
  - `clear()` classmethod resets both `_instance` and `_client` to None
- Import `from openai import OpenAI` at top level (it's lightweight)

**cliniq_v2/modules/m1_ingest.py:**
- Import parse_fhir, parse_text, detect_modality from cliniq.modules.m1_ingest (they are model-agnostic, reuse directly).
- Import ClinicalDocument, DocumentMetadata from cliniq.models.document.
- Write new `parse_image(image_path: str) -> ClinicalDocument` that:
  1. Opens image with `open(path, "rb")`, base64-encodes it
  2. Detects MIME type from suffix (.png -> image/png, .jpg/.jpeg -> image/jpeg)
  3. Calls `OpenAIClient().client.chat.completions.create(model="gpt-4o", ...)` with image_url content and text prompt asking to extract all clinical text
  4. Uses `max_tokens=1024`, `temperature=0.0`
  5. Returns ClinicalDocument with extracted text, source_type="image", modality_confidence=0.90
- Write `ingest()` function that mirrors cliniq.modules.m1_ingest.ingest() but calls the local `parse_image` for image modality, delegating fhir/text to the reused cliniq functions.
- All OpenAI client access inside function bodies only (never at module level).

**cliniq_v2/modules/m2_nlu.py:**
- Import ClinicalEntity, NLUResult from cliniq.models.entities.
- Import CONFIDENCE_THRESHOLD from cliniq_v2.config.
- Define NER_SYSTEM_PROMPT as a string constant with instructions for GPT-4o:
  - Extract clinical entities with fields: text, entity_type (one of "diagnosis", "procedure", "medication", "anatomical_site", "lab_value", "qualifier"), confidence (0.0-1.0), negated (bool), qualifiers (list of strings)
  - Do NOT ask for character offsets (compute post-hoc)
  - Return JSON object: `{"entities": [...]}`
- Write `extract_entities(text: str) -> NLUResult`:
  1. Start timer
  2. Call GPT-4o with `response_format={"type": "json_object"}`, temperature=0.0
  3. Parse JSON response into list of entity dicts
  4. For each entity dict, compute start_char/end_char by searching for entity["text"] in the original `text` using `text.find(entity_text)`. If not found, set start_char=0, end_char=len(entity_text).
  5. Create ClinicalEntity instances (filter by CONFIDENCE_THRESHOLD, but keep qualifiers)
  6. Run qualifier capture logic: separate qualifiers, attach to nearest diagnosis/procedure within 50 chars (reuse pattern from cliniq.modules.m2_nlu.capture_qualifiers)
  7. Return NLUResult(entities=..., processing_time_ms=...)
- Write `process_document(doc: ClinicalDocument) -> NLUResult` convenience wrapper (same as v1).
- Do NOT reuse any d4data-specific functions from cliniq (map_entity_type, extract_raw_entities).
- Negation detection is handled by GPT-4o in the structured output prompt -- no separate detect_negation function needed.
  </action>
